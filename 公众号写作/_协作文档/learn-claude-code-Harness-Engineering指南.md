# 学会Harness Engineering，才算真正懂AI Agent

看了那么多AI Agent教程，动手写的时候还是不知道从哪开始？

我之前也有这个困惑。后来发现问题可能不在教程——大部分教程讲的是"怎么调API"、"怎么写prompt"，但真正决定Agent好不好用的，是另一门手艺：**Harness Engineering**。

**适合谁看**：想从零构建AI Agent的开发者，或者用过Agent框架但想知道底层原理的人

---

## 先说结论

我看了shareAI-lab开源的[learn-claude-code](https://github.com/shareAI-lab/learn-claude-code)项目，最大的收获是搞懂了一件事：

**Agent = Model。代码只是给模型搭的工作台。**

这个工作台叫Harness，包括工具、知识、观测能力、行动接口、权限控制。很多人花时间研究各种Agent框架，却忽略了这套底层能力。这个项目用12节课带你从零搭建，每节课都是可运行的代码，循序渐进。

---

## 什么是Harness Engineering？

说个我踩过的坑：**模型本身就是Agent，真不需要什么框架。**

我一开始也觉得得用个框架才能做Agent。后来发现，Claude、GPT-4这些模型，你给它工具定义，它自己就会决策调用。模型本身就在思考、判断、行动。

那我们写代码干什么？**搭Harness（工作台）**。

Harness包括：
- **Tools** - 给模型用的工具（读文件、跑命令、搜代码）
- **Knowledge** - 项目上下文（CLAUDE.md、文档、代码结构）
- **Observation** - 让模型能看到发生了什么
- **Action** - 让模型能执行操作
- **Permissions** - 控制模型能做什么、不能做什么

这套能力就叫**Harness Engineering**——给AI模型搭建合适的工作环境。

## 12节渐进课程：从循环到自治

这个项目的精华是12节渐进课程（s01-s12），我按顺序说下每节课的核心：

### s01: 基础Agent循环

一切从这开始。说实话，刚看到这段代码时我有点不信——就这么点东西？

核心代码就这几行：

```python
while True:
    response = client.messages.create(
        model="claude-sonnet-4-20250514",
        messages=messages,
        tools=tools
    )

    if response.stop_reason == "end_turn":
        break

    for content in response.content:
        if content.type == "tool_use":
            result = execute_tool(content.name, content.input)
            messages.append({"role": "user", "content": result})
```

就这么简单：发消息 → 检查停止条件 → 执行工具 → 把结果塞回去 → 继续。

很多人觉得Agent神秘，拆开看就是个while循环。我当时跑通这个例子后，突然觉得Agent没那么玄乎了——但这套循环配上工具，就是Agent的雏形。

### s02: 工具使用

s01的循环只能聊天，没啥用。s02给它加上工具，才算是能干活的Agent。

几个关键点：
- 用JSON Schema定义工具参数
- 模型会自己决定什么时候调用、传什么参数
- 你要做的就是执行工具、返回结果

这一课学完，你的Agent就能读文件、跑命令了。我第一次看到Agent自己决定调用工具时，还是有点惊喜的。

### s03: TodoWrite

从这节课开始，Agent有了"记忆"。这是个转折点。

TodoWrite是个特殊工具，让Agent能：
- 记录要做的事
- 标记完成状态
- 在后续对话中引用

这是从"单轮问答"到"多步任务"的关键跨越。没有这个，Agent做不了复杂事。

### s04: 子Agent

复杂任务要拆分，一个人干不过来。这节课教你：
- 启动子Agent处理子任务
- 子Agent完成后返回结果
- 主Agent汇总继续

有点像公司的项目制：老板接单，分给不同小组，最后汇总。但这里"老板"和"小组"都是Agent。

### s05: Skills（技能）

Skills是可复用的能力单元。比如"代码审查"、"写测试"、"重构"。

这一课的核心：
- 把常用操作封装成Skills
- Agent可以根据需要激活不同Skills
- Skills可以组合，形成复杂工作流

### s06: 上下文压缩

这节课我卡得最久。对话长了，上下文会爆，这是Agent处理长任务的硬伤。

这节课解决：

- 识别哪些信息可以压缩
- 保留关键决策和当前状态
- 用摘要替代历史对话

这是Agent能处理长任务的关键能力。理解了这节课，后面的内容才顺畅。

### s07: 任务管理

这节课开始，Agent从"被动响应"变成"主动管理"：

- Agent能追踪多个并行任务
- 自动排优先级
- 遇到阻塞会汇报

### s08: 后台任务

有些任务耗时长，比如跑测试、装依赖，不能阻塞主流程。这节课教你怎么：

- 把任务丢到后台执行
- 主流程继续干别的
- 定期检查后台任务状态

### s09: Agent团队

单Agent能力有限，就像一个人干不了大项目。这节课引入团队：

- 不同Agent有不同专长
- 用协议协调Agent间的协作
- 处理冲突和依赖

### s10: 团队协议

s09搭起了团队，s10定义怎么合作：

- 定义消息格式
- 制定协作规则
- 处理异常情况

### s11: 自治Agent

到这节课，Agent可以独立运行了，你给它一个任务，它能自己搞定：

- 接到任务后自主规划
- 遇到问题自己调整
- 完成后汇报结果

### s12: Worktree隔离

最后一节课解决一个很实际的问题：Agent改代码把项目搞坏了怎么办？

用git worktree解决：
- 每个任务在独立目录跑
- 改坏了也不影响主分支
- 测试通过了再合并

## 为什么强调按顺序学？

这12节课不是随便排的，有明确的依赖关系。我试过跳着看，结果发现前面漏的概念后面都用到了。

```
s01 (循环) → s02 (工具) → s03 (记忆)
                          ↓
            s04 (子Agent) ← s05 (技能)
                    ↓
            s06 (压缩) → s07 (任务管理)
                    ↓
            s08 (后台) → s09 (团队) → s10 (协议)
                                      ↓
                        s11 (自治) → s12 (隔离)
```

跳着学会漏掉关键概念。比如直接看s11的自治Agent，你会发现它用到了s01的循环、s02的工具、s03的TodoWrite、s06的压缩……缺一个都跑不通。

## 学完能干什么？

### 场景一：构建自己的CLI工具

看完s01-s03，你就能做一个能用的命令行助手：
- 理解自然语言指令
- 执行文件操作
- 记住多步任务

### 场景二：代码审查Agent

s04-s05的内容，让你能做一个代码审查Agent：
- 启动子Agent审查不同模块
- 封装"审查"为可复用Skill
- 汇总审查结果

### 场景三：自动化工作流

s06-s08教你处理更复杂的场景：
- 长时间运行的CI/CD任务
- 后台执行的测试
- 多项目并行

### 场景四：多Agent协作系统

s09-s12是进阶内容，适合想搞大项目的：
- 构建多Agent协作平台
- 让AI团队处理复杂项目
- 安全隔离不同任务

## 几个踩坑后的建议

### 建议一：动手跑每节课的代码

别光看代码，一定要跑起来。项目里每个s0X目录都是可运行的：

```bash
cd s01-basic-agent-loop
python main.py
```

看Agent怎么一步步执行，比看十遍代码都有用。我s06卡住就是因为没动手跑，光看代码觉得懂了，一跑就懵。

### 建议二：改代码实验

跑通之后，改改看：
- 加个新工具会发生什么？
- 改变stop_reason的逻辑会怎样？
- 把工具定义写复杂点，Agent能理解吗？

这些实验会暴露很多细节问题。

### 建议三：对比框架实现

学完之后，可以去看看Claude Code、Cursor这些工具的实现。原理其实就是这个循环 + 工具 + Harness，只是封装得更完善。

搞懂底层，用框架才不会迷茫。

### 建议四：别想一口吃成胖子

别上来就想做"全能Agent"。先做个能跑的小东西：
- 自动整理代码格式
- 批量修改文件
- 简单的代码生成

能用了再慢慢加功能。

## 跟LangChain这些框架有啥区别？

| 对比项 | LangChain/AutoGPT | learn-claude-code |
|--------|-------------------|-------------------|
| 学什么 | 框架API | 底层原理 |
| 代码量 | 调库，代码少 | 从零写，代码多 |
| 灵活性 | 受框架限制 | 完全自己掌控 |
| 适用场景 | 快速出原型 | 深度定制 |
| 前置要求 | 会调API | 懂Python |

两种方式各有适用场景：
- 想快速出活 → 用框架
- 想深度理解Agent怎么工作 → 学这个项目

我自己是两个都学了，用框架干活，但遇到问题就回到这个项目找思路。

## 写在最后

我断断续续花了一周时间把这12节课过了一遍（中间卡在s06的上下文压缩好久）。最大的感受是：**Agent开发没什么捷径，就是理解模型 + 搭好Harness**。

那些看起来很酷的Agent演示，背后都是这套基础能力。区别可能就是Harness搭得够不够扎实。

这个项目最宝贵的地方，是它不教你"怎么用框架"，而是教你"框架是怎么实现的"。搞懂这个，用什么框架都顺手。

---

## 总结

learn-claude-code这个项目的核心价值，我觉得就四点：

1. **底层视角** - 从循环开始，理解Agent本质
2. **渐进学习** - 12节课，每节都是前一节的延伸
3. **可运行代码** - 不是纸上谈兵，每行代码都能跑
4. **实战导向** - 学完能直接用在项目里

如果你也好奇"AI Agent到底怎么工作的"，这个项目值得看看。至少对我来说，看完后对Agent的理解比之前清晰多了。
