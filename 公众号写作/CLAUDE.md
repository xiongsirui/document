# 公众号写作工作区

## 第零步：任务识别与路由

### 任务识别机制

#### 1. 工作区判断（第一层）
- **关键词检测**：
  - 检测"公众号"、"商单brief"、"文章" → 公众写作工作区
  - 检测"视频"、"脚本" → 视频创作工作区
  - 检测"Prompt"、"提示词" → Prompt梳理工作区
  - 其他 → 通用工作区

#### 2. 任务类型识别（第二层）
- **A类：新写作（有完整brief）**
  - 识别特征：包含详细的产品信息、功能点、写作要求
  - 处理流程：保存brief → 搜索选题 → 等待选择 → 创作文章 → 审核 → 配图

- **B类：新写作（无brief只有需求）**
  - 识别特征：只有简单需求描述，缺乏详细信息
  - 处理流程：确认需求 → 搜索选题 → 创作文章 → 审核

- **C类：修改已有文章**
  - 识别特征：包含"修改"、"改写"、"调整"等关键词
  - 处理流程：读取原文 → 理解需求 → 执行修改

- **D类：审校/降AI味**
  - 识别特征：包含"太AI味"、"审校"、"检查"等关键词
  - 处理流程：读取原文 → 按checklist检查 → 标注问题 → 提供建议

- **E类：快速咨询**
  - 识别特征：包含"怎么办"、"为什么"、"如何"等疑问词
  - 处理流程：直接回答问题，提供解决方案

### 交互机制
- **选题确认**：提供3-4个选题方向后，必须等待用户选择
- **数据等待**：需要测试数据时，明确说明并等待
- **重要决策**：涉及内容方向、风格选择时，寻求用户确认

## 第二步：公众号调研


# 公众号调研通用工作流程指南


## **一、公众号调研的标准步骤和流程**
1. **明确调研需求**：确定公众号文章的核心主题（如新产品介绍、行业分析、对比评测等），明确需要覆盖的关键信息维度（如产品功能、发布时间、用户评价、竞品对比、安全隐私等）。  
2. **读取写作规则**：获取公众号的写作规范（如风格、结构、字数要求），确保调研内容符合目标平台的输出标准。  
3. **信息收集**：通过多渠道（搜索、官方渠道、第三方平台）获取主题相关的基础信息与深度内容。  
4. **结果整理与保存**：将收集到的信息结构化处理，保存至知识库或文档，便于后续写作调用。  
5. **后续跟进**：记录信息时效性（如更新时间），为后续内容迭代提供依据。  


## **二、搜索的关键词类型和搜索策略**  
### **1. 关键词类型**  
- **核心产品/主题词**：调研对象的核心名称（如“AI浏览器”“2025年新产品”）。  
- **维度扩展词**：围绕核心主题的细分方向（如“功能”“发布”“评价”“对比”“安全隐私”“用户反馈”）。  
- **时间限定词**：明确调研的时间范围（如“2025年”“最新”“近期”），避免过时信息。  
- **平台限定词**：针对特定信息源的关键词（如“Reddit review”“官方博客”）。  

### **2. 搜索策略**  
- **多维度组合**：将核心词与维度词、时间词组合，覆盖全面信息（如“AI浏览器 2025 功能”“AI浏览器 Reddit 评价”“AI浏览器 vs 竞品 对比 2025”）。  
- **分层搜索**：先搜索基础信息（如产品发布、功能介绍），再深入搜索深度内容（如用户评价、安全分析）。  
- **交叉验证**：通过不同关键词组合（如“功能”+“发布”+“评价”）确保信息准确性。  


## **三、信息源的选取原则**  
1. **权威性优先**：优先选择主流媒体（如TechCrunch、CNN、The Washington Post）、行业垂直媒体（如Tom's Guide）或专业机构发布的内容，确保信息可信度。  
2. **官方渠道补充**：获取产品官方博客、官网、社交媒体等第一手信息，避免信息偏差。  
3. **多样性覆盖**：结合第三方平台（如Reddit、知乎）的用户评价，补充真实使用反馈，避免单一视角。  
4. **时效性筛选**：优先选择近期发布的内容（如2025年1月后），确保信息与当前主题匹配。  


## **四、调研结果的处理和保存方法**  
1. **结构化整理**：将信息按维度分类（如“基本信息”“功能特点”“用户评价”“竞品对比”“安全隐私”），形成逻辑清晰的文档。  
2. **规范命名**：保存文件时，文件名包含主题、时间、信息类型（如“AI浏览器调研-2025年10月.md”），便于后续查找。  
3. **记录元数据**：在文档中标注信息收集时间、来源（如“来源：TechCrunch 2025-10-23”）、更新建议（如“下次更新：2025-11-23”），确保信息可追溯。  
4. **知识库保存**：将整理后的文档保存至知识库（如本地文件夹、云文档），方便写作时快速调用。  


## **五、通用调研技巧或注意事项**  
1. **多渠道验证**：对关键信息（如产品功能、发布时间）通过至少2个不同来源交叉验证，避免错误。  
2. **关注时效性**：及时记录信息的发布时间，避免使用过时数据（如2024年的信息用于2025年的文章）。  
3. **优先官方信息**：对于产品核心信息（如功能、发布），以官方渠道为准，第三方信息作为补充。  
4. **保留原始链接**：保存信息时，记录原始来源链接，便于后续核对或补充。  
5. **迭代更新**：定期回顾调研结果，根据主题变化（如产品更新、行业动态）及时补充新信息。  


**总结**：公众号调研的核心是“需求明确→规则适配→多源收集→结构化保存→时效管理”，通过规范流程和策略，确保调研内容全面、准确、符合写作要求，为公众号文章提供可靠的信息支撑。

## 第三步：选题提供


### 通用选题提供工作流程  


#### 1. 基于调研结果提供选题  
- **调研沉淀**：完成目标领域调研后，将调研结果（如行业动态、用户需求、竞品分析等）保存至知识库，为选题生成提供数据支撑。  
- **选题生成逻辑**：从调研中提取关键信息（如热点事件、技术趋势、用户痛点等），基于这些信息生成多个选题方向，每个选题需关联调研结论（如“OpenAI Atlas浏览器24小时被黑”对应调研中的安全漏洞、行业影响等维度）。  


#### 2. 选题的具体方法和标准  
- **选题构成要素**：  
  - **核心理念**：明确选题的核心价值（如“安全警示+风险揭秘”“技术深度分析”等），突出选题的吸引力。  
  - **优势分析**：从时效性（新闻性强、有冲击力）、警示意义（帮助用户规避风险）、技术深度（解决前沿问题）、争议性（引发讨论）等维度评估。  
  - **劣势分析**：评估内容偏向（如偏技术/偏观点）、平衡性（是否客观全面）、技术门槛（用户理解难度）等。  
  - **安全性评估**：判断选题是否涉及安全风险（如漏洞测试），若存在风险则明确“是否需要真实测试”（如“不建议花生亲自测试漏洞”）。  
  - **详细大纲**：拆分选题为章节，明确各部分字数、核心内容（如“开头500字讲事件背景，核心1200字分析漏洞”）。  
- **选题标准**：优先选择**时效性强、吸引力高、工作量可控、内容平衡、安全性合规**的选题。  


#### 3. 选题的分类和类型  
- **分类逻辑**：根据内容属性划分，常见类型包括：  
  - 热点事件类（如产品发布/漏洞事件）；  
  - 技术分析类（如技术原理、解决方案）；  
  - 行业影响类（如趋势预测、竞品对比）；  
  - 用户痛点类（如工具使用、问题解决）。  
- **类型适配**：结合调研目标（如用户需求、行业趋势），选择匹配的选题类型，确保覆盖不同维度。  


#### 4. 与用户互动确认选题  
- **互动方式**：提供选题后，通过**提问/反馈机制**确认用户兴趣（如截图中的“你还敢用吗？”），或根据用户反馈调整选题方向（如优先选择用户关注的热点）。  
- **互动目的**：确保选题符合用户需求，提升选题的针对性和接受度。  


#### 5. 选题相关的技巧或注意事项  
- **时效性优先**：选择近期热点事件，确保选题“新鲜度”，避免过时内容。  
- **吸引力设计**：结合警示、争议、技术深度等元素，强化选题的传播性（如“安全漏洞”“行业冲击”等关键词）。  
- **工作量评估**：明确选题的难度和投入（轻/中/重），帮助用户判断可行性（如“轻度，主要靠调研”）。  
- **安全性合规**：涉及安全测试时，优先选择“模拟分析”而非真实测试，规避风险（如“不建议花生亲自测试漏洞”）。  
- **内容平衡**：通过“优势+劣势”分析，确保选题客观全面，避免片面性。  


通过以上流程，可实现从调研到选题生成、确认、优化的全链路管理，确保选题既基于数据支撑，又贴合用户需求。

## 第五步：内容创作


### 公众号文章创作通用工作流程（第三步：撰写执行）  


#### 1. 基于选题的撰写逻辑  
选题确定后，先锚定**目标风格**（通过学习风格指南类文档，明确写作调性），再从**个人素材库**（如工具使用、行业观察等积累的资料）中提取与选题相关的观点、案例或数据，结合选题核心价值与目标读者需求，启动内容创作。  


#### 2. 文章撰写的具体步骤与方法  
- **风格学习**：阅读“写作参考/风格指南”类文档，快速掌握目标写作风格（如科普风、观点风等），确保内容调性与定位一致。  
- **素材检索**：通过关键词（如“AI工具选择”“Claude使用技巧”）在个人素材库（如浏览器收藏、工具测评记录等）中搜索相关内容，获取真实观点、场景案例或数据支撑。  
- **参考借鉴**：阅读同类公众号的优质文章，分析其结构逻辑、表达方式，为自身创作提供框架参考。  
- **初稿撰写**：基于选题、风格和素材，构建文章框架（标题→核心价值→目标读者→正文→总结），填充内容时突出"核心价值"（如产品分析、场景建议），明确"目标读者"（如AI工具使用者），确保内容有针对性。**初稿保存到** `_协作文档/` 文件夹。  
- **多轮审校**：  
  - 第一轮：**内容审校**，检查逻辑连贯性、数据/事实准确性（如调研来源是否可靠）；  
  - 第二轮：**风格审校**，验证是否符合目标风格（如语言是否口语化、专业度是否匹配）；  
  - （可选）第三轮：**细节优化**，调整段落衔接、语句通顺度，强化个人风格印记。  


#### 3. 内容创作的标准与要求  
- **内容准确性**：数据、案例需来自可靠调研或个人真实经验，避免主观臆断；  
- **价值导向**：明确“核心价值”（如解决用户痛点、提供新视角），确保内容对读者有实际帮助；  
- **读者匹配**：内容需贴合“目标读者”需求（如工具使用者关注实操技巧，而非理论空谈）；  
- **风格一致性**：全文调性统一，符合公众号定位（如科普类需通俗易懂，观点类需逻辑严谨）。  


#### 4. 文章结构与组织方式  
- **标题**：简洁明了，突出核心亮点（如“5个场景告诉你，普通人要不要用Atlas？”）；  
- **核心价值+目标读者**：开篇明确文章能解决什么问题（核心价值）、写给谁看（目标读者）；  
- **正文**：分点/分段展开，逻辑递进（如“场景分析→产品建议→实操技巧”）；  
- **总结/延伸**：收尾强化核心观点，或引导读者互动（如“你用过哪些AI工具？评论区聊聊”）。  


#### 5. 写作技巧与注意事项  
- **个人风格注入**：在内容中融入自身经验或独特视角，避免同质化；  
- **素材真实可信**：优先使用个人实践、调研数据，增强内容说服力；  
- **审校严谨性**：多轮检查避免逻辑漏洞、错别字，确保专业度；  
- **结构清晰化**：用小标题、分段等方式降低阅读门槛，提升可读性。

## 第六步：三遍审校（降AI味专项）

### 6.1 第一遍：内容审校
- **审核重点**：逻辑结构、事实准确性、内容完整性
- **检查项目**：
  - 文章结构是否清晰（标题、开头、正文、结尾）
  - 论点是否明确，论据是否充分
  - 数据和事实是否准确
  - 内容是否完整，无明显遗漏
  - 是否符合公众号写作规范

### 6.2 第二遍：风格审校（降AI味核心）
- **审核重点**：语言风格、可读性、读者体验
- **检查项目**：
  - 语言是否口语化，符合目标读者
  - 段落长度是否适中
  - 是否有小标题引导阅读
  - 专业术语是否解释清楚
  - 是否有个人风格体现
  - 整体调性是否统一

#### 降AI味Checklist
- **去除套话**：
  - 删除"随着科技的发展"
  - 删除"众所周知"
  - 删除"总而言之"
  - 删除其他AI常用套话

- **句式多样化**：
  - 避免连续使用"首先...其次...最后..."
  - 多用短句，避免长难句
  - 改变句式结构，增加节奏感

- **增加个人色彩**：
  - 加入"我觉得"、"我的经验是"
  - 使用具体案例和场景
  - 表达明确的个人观点

- **口语化处理**：
  - 将书面语改为口语
  - 使用日常对话的表达方式
  - 适当使用网络用语（符合公众号调性）

### 6.3 第三遍：细节打磨
- **审核重点**：最终质量把关
- **检查项目**：
  - 错别字和标点符号
  - 段落间距和格式
  - 图片位置和说明
  - 链接有效性
  - 整体阅读体验

### 审核执行方法
- **逐段检查**：按文章结构逐一审核各部分
- **对照标准**：对照公众号写作规范清单
- **标记问题**：用不同标记标注问题类型和位置
- **记录建议**：提供具体的修改建议

## Think Aloud 机制

### 贯穿全程的透明思考
- **每个步骤都要说明思考过程**
- **重要决策要解释原因**
- **遇到问题要说明解决思路**
- **提供选项要说明利弊**

### 核心原则
1. **绝不编造数据**：没有确切来源的数据不使用
