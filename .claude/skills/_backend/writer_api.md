# API写作助手

通过后端服务调用AI模型，提供专业级的写作服务。

## 使用前准备

1. 启动后端服务：
```bash
cd skills
python glm_backend.py
```

2. 确保服务运行在 http://localhost:5000

## 写作功能

### 1. 文章生成
使用GLM-4.6生成高质量文章。

**格式：**
```
[GLM-4.6] 写文章
主题：[文章主题]
风格：[专业/轻松/学术/营销]
字数：[期望字数]
要求：[具体要求]
```

**示例：**
```
[GLM-4.6] 写文章
主题：人工智能在医疗领域的应用
风格：专业
字数：2000
要求：包含具体案例和数据支持
```

### 2. 创意写作
诗歌、故事、剧本等创意内容。

**示例：**
```
[GLM-4.6] 写一首现代诗
主题：科技与自然
长度：30行
风格：自由诗
```

### 3. 商业文案
营销文案、广告语、产品描述等。

**示例：**
```
[GLM-4.6] 写营销文案
产品：智能家居系统
目标客户：年轻家庭
卖点：便捷、节能、安全
类型：社交媒体推广
```

### 4. 学术写作
论文摘要、研究报告、学术综述等。

**示例：**
```
[GLM-4.6] 写论文摘要
研究领域：机器学习
标题：深度学习在图像识别中的应用
字数：300
期刊：IEEE Transactions
```

### 5. 技术文档
API文档、用户手册、教程等。

**示例：**
```
[GLM-4.6] 写API文档
API名称：用户认证接口
功能：JWT token验证
包含内容：使用示例、错误码、注意事项
```

## 高级功能

### 内容改写
保持原意，改变风格或结构。

```
[GLM-4.6] 改写内容
原文：[粘贴原文]
目标风格：[简洁/正式/通俗]
目标长度：[增减要求]
```

### 内容扩展
对简短内容进行详细展开。

```
[GLM-4.6] 扩展内容
大纲：
1. 主题介绍
2. 核心观点
3. 具体案例
4. 总结

请扩展成完整文章
```

### 多版本生成
为同一主题生成不同版本。

```
[GLM-4.6] 生成3个版本
主题：环保的重要性
版本1：面向政府报告
版本2：面向公众宣传
版本3：面向教育材料
```

## 成本控制

### 模型选择
- **GLM-4.6**：最高质量，适合重要内容
- **GLM-4.5**：平衡性价比
- **GLM-3-Turbo**：最经济，适合草稿

### 优化技巧
- 明确要求减少重写次数
- 使用大纲先构思再写作
- 批量处理降低单位成本

## 常见问题

**Q: API调用失败？**
A: 检查后端服务是否运行，确认GLM_API_KEY已设置

**Q: 生成内容不满意？**
A: 尝试：
- 更详细的要求描述
- 调整temperature参数
- 指定具体的风格要求

**Q: 成本太高？**
A: 建议：
- 使用GLM-4.5或GLM-3-Turbo
- 先生成大纲再展开
- 批量处理相似任务
