# 公众号写作实战教程

手把手教你使用这套工作流写出高质量公众号文章。

---

## 开始之前

你只需要准备：
1. 一个想写的主题（哪怕很模糊也行）
2. 一些你的真实经历/观点（如果有的话）

---

## 第一步：告诉我你想写什么

直接说你的需求，比如：

```
写一篇关于AI编程工具的公众号文章
```

或者更具体一点：

```
写一篇公众号文章
主题：Claude Code使用体验
目标读者：程序员
字数：3000字左右
```

**我会做什么：**
- 判断这是公众号写作任务
- 判断任务类型（新写作/修改/审校）
- 开始执行对应流程

---

## 第二步：等我搜索信息（如果需要）

如果涉及新产品、新技术，我会先搜索最新信息。

**你会看到：**
```
[思考] 这个主题涉及2024年的新工具，我的训练数据可能不是最新的。
[行动] 让我先搜索一下最新信息...
[结果] 找到以下信息：
- Claude Code 最新版本：xxx
- 主要功能：xxx
- 信息来源：官方文档
```

**你需要做什么：**
- 确认信息是否准确
- 补充我可能遗漏的信息

---

## 第三步：选题讨论（重要！）

我会提供 3-4 个选题方向供你选择。

**你会看到：**
```
## 选题方案

### 选题1：Claude Code深度评测
- 核心角度：功能对比 + 真实使用体验
- 工作量：⭐⭐⭐
- 优势：有真实数据，可信度高
- 劣势：需要你提供测试数据
- 大纲：
  • 开头（200字）：为什么要用AI编程工具
  • 主体（2000字）：功能介绍 + 对比测评
  • 结尾（300字）：使用建议

### 选题2：从Cursor到Claude Code：我的迁移之路
- 核心角度：个人体验 + 迁移过程
- 工作量：⭐⭐
- 优势：故事性强，有共鸣
- 劣势：主观性较强
...

请选择一个选题，或告诉我你的想法。
```

**你需要做什么：**
- 选一个你喜欢的（直接说"选题1"或"我选第一个"）
- 或者告诉我你想调整的方向

---

## 第四步：提供你的真实素材（关键！）

这一步决定了文章是否有"人味"。

**我会问你：**
```
为了让文章更真实，请提供一些你的真实经历：
1. 你是什么时候开始用这个工具的？
2. 遇到过什么问题？怎么解决的？
3. 有没有印象深刻的使用场景？
4. 你对这个工具的真实评价是什么？
```

**你需要做什么：**
- 随便聊聊你的真实感受
- 不需要写得很正式，口语化就行
- 越具体越好（具体时间、具体场景、具体数字）

**示例回答：**
```
我大概用了两周。一开始觉得挺惊艳的，写Python代码几乎不用改。
但后来发现它对复杂业务逻辑理解不太好，有一次让它重构一个支付模块，
改出了bug，排查了两小时。总体来说7分吧，日常写代码够用，
复杂项目还是要自己把关。
```

---

## 第五步：我开始写初稿

基于你选的选题和提供的素材，我会写初稿。

**你会看到：**
- 完整的文章草稿
- 我会标注哪些地方用了你的素材

**你需要做什么：**
- 读一遍，看看方向对不对
- 告诉我哪里需要调整

---

## 第六步：三遍审校（降AI味）

这是最关键的一步。我会进行三遍审校：

### 第一遍：内容审校
```
检查中...
- [x] 事实准确
- [x] 逻辑清晰
- [x] 无编造数据
发现问题：第3段的版本号需要确认
```

### 第二遍：风格审校（降AI味）
```
修改前：
"在当今AI技术飞速发展的时代，编程工具也在不断进化..."

修改后：
"Claude Code出了。我用了两周，确实比Cursor好用。"

---
修改前：
"通过充分利用Claude Code的能力，可以显著提升开发效率..."

修改后：
"用好Claude Code，开发速度能快不少。我测下来，平均每个项目省3-5小时。"
```

### 第三遍：细节打磨
```
检查中...
- 句子长度：OK（平均20字）
- 段落长度：OK（手机3-5行）
- 节奏变化：调整了第4段，加入短句
```

**你需要做什么：**
- 可以直接说"继续"让我完成审校
- 或者指出你觉得需要保留的表达

---

## 第七步：配图建议

我会给出配图建议：

```
## 配图建议（共5张）

1. 【封面图】Claude Code界面截图 + 标题文字
   - 位置：文章开头
   - 建议：用官方截图，加上你的评分

2. 【对比图】Claude Code vs Cursor 功能对比表
   - 位置：第二部分
   - 建议：做一个简单的表格截图

3. 【代码截图】实际使用效果
   - 位置：第三部分
   - 建议：截一个你觉得惊艳的代码生成效果

4. 【数据图】效率提升数据
   - 位置：第四部分
   - 建议：可以用简单的柱状图

5. 【结尾图】关注引导
   - 位置：文章结尾
```

**你需要做什么：**
- 准备这些配图
- 或者告诉我哪些图你做不了，我帮你想替代方案

---

## 第八步：最终定稿

我会输出：
1. 完整的 Markdown 格式文章
2. 配图位置标注
3. 发布前检查清单

**你需要做什么：**
- 复制到公众号后台
- 上传配图
- 发布！

## 实战示例

### 示例对话

**你：** 写一篇公众号文章，主题是远程办公工具

**我：**
```
[公众号] 已识别：公众号写作任务
[任务类型] B - 新写作任务（需先了解需求）

在开始之前，我想了解几个问题：
1. 目标读者是谁？（职场人/创业者/自由职业者）
2. 想重点介绍哪些工具？
3. 你有使用这些工具的真实经历吗？
```

**你：** 目标读者是职场人，重点介绍飞书和Notion，我两个都用过

**我：**
```
好的，让我先搜索一下这两个工具的最新信息...

[搜索完成]

现在提供3个选题方案：

选题1：飞书 vs Notion：远程办公该选哪个？
选题2：我用飞书和Notion一年后的真实感受
选题3：远程办公工具选择指南：从需求出发

你倾向哪个？
```

**你：** 选题2，我有很多真实感受可以分享

**我：**
```
太好了！请分享一下你的真实经历：
1. 你是什么场景下开始用这两个工具的？
2. 各自最让你满意和不满意的地方是什么？
3. 有没有具体的使用场景可以分享？
```

**你：** [分享真实经历...]

**我：** [开始写作...] → [三遍审校...] → [输出定稿]

## 常见问题

**Q：我没有什么真实经历怎么办？**
A：那就聚焦在"信息整合"类文章，比如"2024年最值得关注的5个AI工具"，这类文章不需要太多个人经历。

**Q：我想快点写完，能跳过步骤吗？**
A：可以，直接说"跳过选题讨论"或"直接写初稿"。但我会提醒你风险。

**Q：写完的文章还是有AI味怎么办？**
A：把文章发给我，说"帮我降AI味"，我会专门做一轮风格审校。

**Q：我只想要个大纲/选题怎么办？**
A：直接说"只要大纲"或"只要选题"，我不会写完整文章。
