# 个人素材库搭建指南

## 方式1：轻量版素材库（无需后端）

使用 Markdown 文件管理素材，简单高效。

### 1. 创建素材目录结构

```
d:\code\ai\document\素材库\
├── 2025\
│   ├── 01-一月\
│   ├── 02-二月\
│   └── 12-十二月\
├── 主题\
│   ├── AI工具\
│   ├── 编程技术\
│   ├── 产品管理\
│   └── 职场心得\
└── 索引.md
```

### 2. 使用文件模板创建素材

```
[AI] 创建素材
类型：[经历/评价/案例/洞察/数据]
内容：[素材内容]
标签：[标签1, 标签2]
保存到：素材库/2025/12/
```

### 3. 搜索素材

```
[AI] 搜索素材
关键词：[关键词]
目录：素材库/
时间范围：[最近30天]
```

[素材内容]

### 3. 使用 Dataview 查询

```markdown
```dataview

  category as "类型",
  date as "日期",
  credibility as "可信度"
FROM "素材库"
WHERE contains(tags, "AI工具")
SORT date DESC
```

## 方式4：使用 AI Skill 管理（最简单）

### 添加素材
```
[AI] 添加素材
类型：产品评价
内容：今天测试了XX工具...
标签：[AI工具, 测试]
```
系统会自动：
- 保存到素材库/2025/12/
- 添加标签和元数据
- 更新索引文件

### 搜索素材
```
[AI] 搜索素材
关键词：AI工具
类型：评价
时间：最近30天
```

## 实际使用流程

### 日常积累
1. **随手记录**：遇到有价值的经历立即记录
2. **定期整理**：每周整理一次素材
3. **标签系统**：建立自己的标签体系

### 写作时调用
1. **搜索相关素材**
2. **选择合适的案例**
3. **融入文章**

## 标签体系建议

### 按类型
- #经历 - 个人体验
- #评价 - 产品/服务评价
- #案例 - 成功/失败案例
- #洞察 - 思考和总结
- #数据 - 数字和统计

### 按主题
- #AI工具
- #编程技术
- #产品管理
- #职场心得
- #学习方法

### 按可信度
- #可信-10 - 亲身经历
- #可信-8 - 有据可查
- #可信-5 - 道听途说

## 素材内容结构

好的素材应该包含：
1. **具体事实**：什么时间、什么地点、发生了什么
2. **个人感受**：当时的感受和想法
3. **结果影响**：带来了什么变化
4. **反思总结**：学到了什么

## 示例素材

```markdown
---
type: 素材
category: 评价
tags: [Claude Code, AI编程, 测试]
date: 2024-12-15
source: 个人测试
credibility: 9
---

## 测试Claude Code两周的体验

### 背景
作为程序员，一直在寻找能提升编码效率的AI工具。

### 过程
- 第1周：主要用于代码生成和优化
- 第2周：尝试复杂项目的重构

### 发现
1. Python代码生成准确率95%以上
2. 能理解复杂的业务逻辑
3. 但UI交互还需要改进

### 结果
编码效率提升约40%，bug率下降30%
```
