# 完整写作工作流

实现reference.md中的9步专业写作流程，提供结构化的内容创作解决方案。

## 工作流程

使用以下命令启动完整工作流：

```
[Workflow] 开始写作流程
主题：[文章主题]
类型：[公众号/博客/报告]
目标字数：[字数]
特殊要求：[其他要求]
```

```

- 融入真实案例

- 加入个人视角

```

## Step 8: 三遍审校

### 8.1 内容审校
```
[Workflow] Step8.1 - 内容审校
检查项：
□ 事实准确（数据、时间、产品名称）
□ 逻辑清晰（前后无矛盾）
□ 结构合理（无跑题）
□ 无编造（所有数据和案例都真实）
```

### 8.2 风格审校（降AI味）
```
[Workflow] Step8.2 - 风格审校
AI味检查：
□ 删除套话（"在当今时代"、"综上所述"）
□ 拆解AI句式（"不是…而是…"）
□ 替换书面词（"显著提升"→具体数字）
□ 口语化改写（"进行操作"→直接动词）
□ 加入真实细节
□ 加入个人态度
```

### 8.3 细节打磨
```
[Workflow] Step8.3 - 细节打磨
检查项：
□ 句子长度（15-25字为主）
□ 段落长度（手机3-5行）
□ 标点自然（多用句号）
□ 节奏变化（快慢结合）
```

## Step 9: 文章配图

```
[Workflow] Step9 - 配图建议
基于文章内容，建议配图：

必需配图（5张）：
1. [位置：开头] - [描述：概念图]
2. [位置：第2部分] - [描述：数据图表]
3. [位置：第3部分] - [描述：应用场景]
4. [位置：第4部分] - [描述：对比图]
5. [位置：结尾] - [描述：总结图]

图片来源优先级：
1. 公共领域（Wikimedia Commons）
2. AI生成（火山引擎）
3. 免费图库（Unsplash）
```

## 使用技巧

1. **逐步执行**：不要跳过前面的步骤，每步都认真完成
2. **保持真实**：所有案例和数据都必须真实可验证
3. **积累素材**：定期更新个人素材库
4. **记录经验**：保存成功的Brief和选题

## 注意事项

- 整个流程可能需要30-60分钟
- 建议分批次完成，避免一次性处理太多内容
- 真实性是第一原则，宁可内容简单也不要编造
- 保持个人风格，避免千篇一律
