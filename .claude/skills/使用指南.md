# AI Skills 详细使用指南

本指南将详细说明如何在实际工作中使用这些Skills，以及它们如何协同工作。

## 📋 Skills 概览

### API驱动的Skills（需要后端服务）
- **writer_api.md** - 主要的写作工具，调用AI API生成内容
- **content_analyzer.md** - 分析文本质量、风格、情感等
- **code_assistant.md** - 生成、解释和优化代码
- **translator.md** - 多语言翻译
- **knowledge_manager.md** - 知识管理和学习辅助
- **knowledge_base.md** - 个人素材库管理
- **writing_workflow.md** - 完整的9步写作流程

### 纯模板的Skills（无需API）
- **article_writer.md** - 基于模板快速生成结构化内容
- **writer_template.md** - 模拟特定风格的写作

## 🚀 快速上手

### 1. 安装和配置后端服务

```bash
# 1. 安装依赖
pip install flask zhipuai

# 2. 设置API Key（以GLM为例）
export API_KEY="your-glm-api-key"

# 3. 启动后端服务
cd skills
python glm_backend.py

# 服务将在 http://localhost:5000 运行
```

### 2. 在Claude Code中使用

后端启动后，你可以直接在对话中使用各种技能：

```
用户：帮我写一篇关于人工智能在医疗领域应用的文章

Claude：[AI] 我来为您写一篇关于人工智能在医疗领域应用的文章

[AI将调用writer_api技能生成文章]
```

## ⚠️ 注意事项

### 1. API限制
- 确保后端服务正常运行
- 注意API调用频率限制
- 监控使用成本

### 2. 内容质量
- AI生成的内容需要人工审核
- 重要信息需要验证真实性
- 避免完全依赖AI生成

### 3. 版权和隐私
- 不要将API返回的内容直接用于商业用途
- 个人知识库中的素材要注意隐私保护
- 遵守相关法律法规

## 🔧 故障排除

### 后端服务连接失败
```bash
# 检查服务是否运行
curl http://localhost:5000/health

# 查看服务日志
python glm_backend.py
```

### API调用失败
```
1. 检查API Key是否正确
2. 验证网络连接
3. 查看API配额
```

### 技能不响应
```
1. 确认后端服务已启动
2. 检查后端URL是否正确
3. 重启Claude Code
```
